# АИС «Электронная библиотека»

## Важное по рецензиям

По заданию рецензии могут писать все три авторизованные роли: `Пользователь`, `Модератор` и `Администратор`. В проекте это реализовано явно: кнопка «Написать рецензию» появляется для любой из этих ролей, если пользователь ещё не оставлял рецензию на выбранную книгу. Любая новая рецензия, независимо от роли автора, получает статус `На рассмотрении` и появляется в общем списке на странице книги только после одобрения модератором или администратором.


Flask-приложение для ведения реестра книг и читателей, поиска книг, добавления рецензий и модерации рецензий по индивидуальному варианту 1.

## Что реализовано по заданию

- Главная страница со списком книг, сортировкой по году выхода: сначала новые.
- Пагинация: не более 10 книг на странице.
- Поиск книг по названию, автору, издательству, описанию и жанру.
- Просмотр книги доступен всем пользователям.
- Добавление книги доступно только администратору.
- Редактирование книги доступно администратору и модератору.
- Удаление книги доступно только администратору через модальное окно подтверждения.
- При удалении книги через `ON DELETE CASCADE` удаляются рецензии, связи книги с жанрами и запись обложки.
- Файл обложки удаляется из файловой системы, если он больше не используется другими книгами.
- Форма добавления/редактирования книги реализована через Jinja-макрос.
- При добавлении книги загружается обязательная обложка через `enctype="multipart/form-data"`.
- При редактировании книги поле загрузки обложки отсутствует.
- Описание книги и текст рецензии поддерживают Markdown.
- Markdown-текст проходит санитайзер Bleach перед сохранением и после преобразования в HTML.
- Обложки хранятся в отдельной таблице `covers` с полями `filename`, `mime_type`, `md5_hash`, `book_id`.
- Для одинаковых изображений повторный файл не сохраняется: используется уже существующее имя файла по MD5.
- Рецензии доступны авторизованным пользователям всех трёх ролей.
- Пользователь может оставить только одну рецензию на одну книгу.
- Новая рецензия автоматически получает статус `На рассмотрении`.
- На странице книги отображаются только одобренные рецензии.
- Если пользователь уже писал рецензию, вместо кнопки показывается его рецензия и её статус.
- Для роли `Пользователь` есть раздел `Мои рецензии`.
- Для роли `Модератор` есть раздел `Модерация рецензий`; администратор также имеет доступ как полный пользователь системы с пагинацией и рассмотрением заявок.
- Страница входа содержит логин, пароль, кнопку `Войти` и чекбокс `Запомнить меня`.
- При неверном логине/пароле выводится сообщение: `Невозможно аутентифицироваться с указанными логином и паролем`.
- При попытке открыть закрытую страницу без входа выводится сообщение: `Для выполнения данного действия необходимо пройти процедуру аутентификации`.
- При недостатке прав выводится сообщение: `У вас недостаточно прав для выполнения данного действия`.
- В базовом шаблоне указаны группа и ФИО.

## Быстрый запуск на macOS

### 1. Перейти в папку проекта

```bash
cd ~/Downloads/library
```

Если папка называется `library 2`, используй кавычки:

```bash
cd ~/Downloads/"library 2"
```

### 2. Установить MySQL и системные зависимости

```bash
brew install python@3.12 pkg-config mysql mysql-client
brew services start mysql
```

### 3. Создать базу и пользователя MySQL

```bash
mysql -u root
```

Внутри MySQL выполнить:

```sql
CREATE DATABASE library_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
DROP USER IF EXISTS 'library_user'@'localhost';
CREATE USER 'library_user'@'localhost' IDENTIFIED BY 'library_pass';
GRANT ALL PRIVILEGES ON library_db.* TO 'library_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 4. Создать виртуальное окружение

```bash
/opt/homebrew/bin/python3.12 -m venv venv
source venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
```

### 5. Установить зависимости Python

```bash
export PATH="/opt/homebrew/opt/mysql-client/bin:$PATH"
export LDFLAGS="-L/opt/homebrew/opt/mysql-client/lib"
export CPPFLAGS="-I/opt/homebrew/opt/mysql-client/include"
export PKG_CONFIG_PATH="/opt/homebrew/opt/mysql-client/lib/pkgconfig"

pip install -r requirements.txt
```

### 6. Инициализировать БД

```bash
python init_db.py
```

Внимание: `init_db.py` пересоздаёт таблицы по `schema.sql`, то есть старые данные в учебной базе будут удалены.

### 7. Запустить приложение

```bash
python app.py
```

Открыть в браузере:

```text
http://127.0.0.1:5000
```
